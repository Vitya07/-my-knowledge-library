# Бинарный поиск

## Почему так называется?
«Бинарный» — от слова «два». Алгоритм каждый раз делит массив на две части и выбирает одну из них.

## Когда использовать?
✅ Массив **отсортирован**  
✅ Нужно быстро найти **индекс** элемента  

❌ Не использовать, если массив не отсортирован.

## Пример на Go
```go
func binary_search(arr []int, item int) int {
	startPos := 0
	endPos := len(arr) - 1

	for startPos <= endPos {
		middlePos := (startPos + endPos) / 2
		finderNum := arr[middlePos]

		if finderNum == item {
			return middlePos
		} else if finderNum > item {
			endPos = middlePos - 1
		} else {
			startPos = middlePos + 1
		}
	}
	return -1
}
