# Бинарный поиск

## Почему так называется?
«Бинарный» — от слова «два». Алгоритм каждый раз делит массив на две части и выбирает одну из них.

## Когда использовать?
✅ Массив **отсортирован**  
✅ Нужно быстро найти **индекс** элемента  

❌ Не использовать, если массив не отсортирован.

## Пример на Go
```go
func binary_search(arr []int, item int) int {
	startPos := 0
	endPos := len(arr) - 1

	for startPos <= endPos {
		middlePos := (startPos + endPos) / 2
		finderNum := arr[middlePos]

		if finderNum == item {
			return middlePos
		} else if finderNum > item {
			endPos = middlePos - 1
		} else {
			startPos = middlePos + 1
		}
	}
	return -1
}
```

# eetcode: 584. Find Customer Referee


+-------------+---------+
| Column Name | Type    |
+-------------+---------+
| id          | int     |
| name        | varchar |
| referee_id  | int     |
+-------------+---------+
In SQL, id is the primary key column for this table.
Each row of this table indicates the id of a customer, their name, and the id of the customer who referred them.
 

Find the names of the customer that are either:

referred by any customer with id != 2.
not referred by any customer.
Return the result table in any order.

The result format is in the following example.


```sql
SELECT 
    name
FROM 
    Customer
    
WHERE referee_id != 2 OR referee_id IS NULL;

```


# Leetcode: 1757. Recyclable and Low Fat Products


```sql
SELECT
    product_id
FROM

    low_fats = 'Y' AND recyclable = 'Y'
```
